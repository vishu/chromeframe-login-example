Overview

This application demonstrates a problem I'm having with Chrome Frame logging into my application, thinklinkr.com

Flow

the flow is supposed to be as follows:

  1. START AT the root of the application, in this case ("/redirect")
  2. click the login link to go to the login page
  3. click the button to post the login form
  4. get redirected to the root ("/redirect")
  5. get redirected to the logged-in page ("/redirect/login") because you are logged in.
but chrome frame never seems to follow the first redirect after login

Notes

  1. the source of this rails app is available at git hub at http://github.com/vishu/chromeframe-login-example
  2. this example is probably more complicated than necessary and I will pare down the functionality to help identify the problem as time permits
  3. when I run the example in development, the last message I see is that the 302 redirect is sent after posting the login form
